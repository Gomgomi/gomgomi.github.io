Programmable Search Engine 사용하여 검색결과 획득 해보기

데이터 수집에 대해 공부 중 검색에 흔히 사용하는 Google을 이용하여 검색 결과를 받을 수 있는 방법이 없을까 알아보다가 Programmable Search Engine(또는 Custom Search Engine)을 알게되어 관련내용을 정리해보았다.

예전에 크롤링 검색하면 흔히 나오는 BeautifulSoup4 라이브러리를 사용하여 검색 결과를 끌어모은적이 있었지만, 정상적인 방법이 아닌 것 같아 구글에서 제공하는 기능을 사용해보기로 했다.

Programmable Search Engine - Guide의 튜토리얼을 따라한 내용이며, 어려운 내용이 아니라서 쉽게 따라할 수 있었다.

Control Panel에서 Programmable Search Engine 생성

먼저 검색엔진을 생성해보자.
Control Panel로 이동하여 Search engine을 사용할 구글 계정으로 로그인한다.

검색할 사이트 항목에 검색에 사용되는 도메인을 입력한다.
도메인 입력 형식은 설명에도 나와있듯 서브도메인이나 특정도메인 등 자유롭다.

검색 엔진 이름 입력란에 검색엔진의 이름을 적어주고 ‘만들기’ 버튼을 눌러 검색엔진을 생성한다.

종종 웹 페이지나 블로그에서 내부에 구글 검색 창을 가지고 있는 것들이 있는데, 여기서 생성한 검색 엔진을 페이지에 추가한 형태이다. ‘검색엔진 수정’에서 본인에게 검색 범위나 디자인 등을 수정하여 사용할 수 있다.

API 사용준비

앞서 생성한 검색엔진을 API 호출을 통해 검색 결과를 획득해보자.
동작확인을 위해 최소한으로 필요한 parameter는 cx, key, q 이렇게 세 가지이다.
이제 이 세 가지를 하나하나 확인해보자!

cx - Search Engine ID

Control Panel페이지의 왼쪽 메뉴에서 ‘검색엔진 수정’의 콤보박스에서 앞서 생성한 검색 엔진을 선택하면 검색 엔진 ID를 확인할 수 있다.

key - API key

Custom Search JSON API: Introduction 문서에서 API key를 획득 할 수 있다.

페이지로 이동하면 눈에 딱 띄는 파란색 ‘Get a Key’ 버튼이 있다.
이 버튼을 누르면 프로젝트를 선택하라고 하는데, 기존에 생성했던 프로젝트가 있으면 선택하면 되고 없다면 새로 생성하면 된다.
프로젝트를 선택하거나 생성하고 나면 API key가 생성된다. API key는 저장해두고 필요할 때 사용하도록 하자.



q - Query

q 파라미터는 간단히 검색어라고 생각하면된다.

혹시 다른 파라미터도 궁금하다면 Method: cse.list에서 확인할 수 있다.

API 사용하여 결과 획득

API를 사용할 준비가 다 되었으니, 이제 사용을 해보자!
직접 코드를 작성하거나 curl 툴을 사용해도 되지만 구글에서는 쉽게 테스트 할 수 있도록 웹에서 툴을 제공하고 있다.

Custom Search JSON API: Introduction 문서의 가장 아래부분에 “Try this API” tool을 클릭하면 툴 화면으로 이동한다.

툴 화면의 왼쪽 파라미터 목록중 cx와 q 파라미터 값을 입력해준다. key 값은 API key를 획득했으면 자동으로 넣어주는 듯 하다.
각 파라미터 값을 입력하고 파라미터 입력 아래부분에 ‘EXECUTE’ 버튼을 누르면 요청 결과를 오른쪽에서 확인할 수 있다.

검색 결과는 response body로 JSON 포맷으로 수신되며 “items”에 해당하는 내용이 검색된 결과 값들이다.
이 중 각자 필요한 필드를 사용하면 된다.
